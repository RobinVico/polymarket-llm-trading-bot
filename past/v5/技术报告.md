### 2.3 日常监控

- Dashboard 永远开着, 每 30s 自动刷新仓位/价格/盈亏
- 看 4 个 metric 卡: 总盈亏 / 持仓数 / 总投入+总收入 / 资产组合+现金
- 看 🎯 事件中心: 实时榜/涨跌榜/现值榜切换
- 看每个仓位的决策状态 badge:
  - 🟢 `HOLD` — 继续放着
  - 🟡 `MARGINAL` / `SOFT_NEGATIVE` — 需要重评
  - 🔴 `AT_TARGET` — 点 badge 走 sell
  - 🟠 `FROZEN` (v5 新) — 急跌冻结中, bot 不评估止损 (等回弹或到期); 可点旁边 🔄 重评按钮手动重审 q
  - 灰 `PENDING` / `NO_META` — 还没填 q

### 2.4 重评流程

| 步 | 在哪 | 做什么 |
|---|---|---|
| 1 | 仓位 "🔄 重评" 按钮 | 打开 reeval-panel |

| 5 | Dashboard 选 | 按结果点 "维持 q" / 填新 q "更新 q" / "建议清仓" |

### 2.5 加仓 / 清仓 (v5 可直接在 dashboard 操作)

每个仓位的决策状态行末尾:

```
[$ 输入框] [＋加仓 绿]  [✗清仓 红]
```

- **加仓**: 输入美元金额 → 点 ＋加仓 → 弹 confirm "买 X 股 @ Y%, 约 $Z" → executor.buy() FAK
- **清仓**: 点 ✗清仓 → 弹 confirm "卖全 X 股 @ Y%" → executor.sell() 全平

**Bot 自动卖** 在 v5.1 这 6 个状态触发: `TAKE_PROFIT_PRICE` / `TAKE_PROFIT_PNL` / `TIME_STOP` / `SLOW_DROP` / `FROZEN_EXPIRED` / `ABSOLUTE_FLOOR`。其他都需用户确认。

## 四、决策引擎 (v5.1 核心)

### 4.1 优先级 (第一个匹配胜出)

`modules/monitor.py:_evaluate_position()`:

| 优先级 | 状态 | 触发条件 | 动作 |
|---|---|---|---|
| 0 | `NO_META` | meta 行不存在 | 跳过 |
| **1a** | **`TAKE_PROFIT_PRICE`** | **cur ≥ 90¢** (v5.1 新增) | **自动全卖** |
| **1b** | **`TAKE_PROFIT_PNL`** | **(cur−entry)/entry ≥ +100%** (v5.1 新增) | **自动全卖** |
| 2 | `TIME_STOP` | 距结算 ≤ 2 天 且 \|p − entry\| < 5pp | **自动卖** |
| 3 | `FROZEN` 检查 | 在冻结期 且 价未回 entry−10pp 内 | 不评估其他, 不卖 |
| 3 | `FROZEN_EXPIRED` | 冻结期满 + 价格仍 < stop | **自动卖** |
| 4 | `ABSOLUTE_FLOOR` | cur/entry < 40% | **自动卖** (兜底) |
| 5 | `SLOW_DROP` | cur ≤ stop(entry) 且跌速 ≥ 30min | **自动卖** |
| 5 | `FROZEN_FRESH` | cur ≤ stop(entry) 且跌速 < 30min | 写状态 + 入冻结期 |
| 6 | Edge-based | `edge = q − p` | 等用户确认 |

**v5.1 删除的状态**:
- `BLACKSWAN_HEDGE` (原 p ≥ 97% 减半提示) — 被 `TAKE_PROFIT_PRICE` (90¢ 自动全卖) 完全覆盖且更早触发, 已无意义。
- `CONFIRMED_NEGATIVE` (原 edge < -3pp 第 2 次重评) — 简化为 `SOFT_NEGATIVE` 一档, 不再区分重评次数。`db.position_meta.last_q_update_with_negative_edge` 字段保留 (兼容老数据, 不再读)。

**v5.1 止盈规则设计动机**:
- **90¢ 价格止盈**: 90¢ 以上继续涨的边际收益很低 (年化约 25%), 但流动性变差 + UMA 结算风险变大。历史教训: Trump publicly insult 涨到 100¢ 没卖, 后被止损卖在 13¢。
- **+100% 浮盈翻倍止盈**: 翻倍是"已经赚到一倍"的心理/数学双重锚点, 后续回吐就是绝对损失。+70% 太早 (会过早锁住 Trump AI export +131%、Gemini 3.2 +103%、Hormuz 0-10 +76% 这类大赢家), +100% 更合理。
- **为什么放最高优先级**: 止盈是无条件的"该跑了"信号, 不应该被任何 edge 重评或冻结状态压住。

#### Edge-based 子状态 (优先级 6 内)

| 子状态 | 条件 | 推荐动作 |
|---|---|---|
| `HOLD` | `edge > +2pp` | 持有 |
| `MARGINAL` | `−3pp ≤ edge ≤ +2pp` | 观察 |
| `AT_TARGET` | `edge < −3pp` 且未重评 | 建议清仓 |
| `SOFT_NEGATIVE` | `edge < −3pp` + 重评过 | 重评 / 决定 |

### 4.2 阈值常量 (modules/monitor.py 顶部)

```python
# Edge-based
HOLD_MIN_EDGE_PP = 2.0
SOFT_NEGATIVE_THRESHOLD_PP = -3.0

# === v5.1 自动止盈 (最高优先级, 触发即全卖) ===
TAKE_PROFIT_PRICE = 0.90     # cur ≥ 90¢ → 全卖
TAKE_PROFIT_PNL_PCT = 1.00   # 浮盈 ≥ +100% (翻倍) → 全卖

# TIME_STOP (跟 v4 一致)
TIME_STOP_DAYS = 2
TIME_STOP_DRIFT_PP = 5.0

# === v5 止损 ===
SLOW_DROP_MIN_MINUTES = 30   # 跌破 stop 持续 >30min → 慢跌卖
FREEZE_DURATION_HOURS = 24   # 急跌冻结时长
UNFREEZE_RECOVERY_PP = 10    # 冻结期价回到 entry−10pp 内 → 解冻
ABSOLUTE_FLOOR_PCT = 0.40    # cur/entry < 40% → 兜底卖

# 心跳
CHECK_INTERVAL = 180
SNAPSHOT_INTERVAL = 1800

# Legacy compat (dashboard 老 import, 不影响决策)
DISASTER_DROP_PP = 25.0
```

### 4.3 stop_price 分档函数

`modules/monitor.py:_stop_price(entry)`:

| entry | drop | stop 示例 |
|---|---|---|
| ≥ 50¢ | 25pp | 70¢→45¢ / 50¢→25¢ |
| 30-50¢ | 18pp | 40¢→22¢ / 30¢→12¢ |
| 15-30¢ | 10pp | 20¢→10¢ / 17¢→7¢ |
| < 15¢ | — | None (走 ABSOLUTE_FLOOR 兜底) |

设计动机: 低价仓位本来就没多少空间往下跌, drop pp 越小越合理。这解决了 v4 单 25pp 阈值在低价位失效的问题 (17¢ 入场无法跌 25pp)。

### 4.4 自动 vs 手动 vs 冻结

| 状态 | 动作 |
|---|---|
| `TAKE_PROFIT_PRICE` | **bot 自动全卖** (v5.1, 最高优先级) |
| `TAKE_PROFIT_PNL` | **bot 自动全卖** (v5.1, 最高优先级) |
| `TIME_STOP` | **bot 自动 sell** |
| `SLOW_DROP` | **bot 自动 sell** |
| `FROZEN_EXPIRED` | **bot 自动 sell** (24h 后未恢复) |
| `ABSOLUTE_FLOOR` | **bot 自动 sell** (兜底) |
| `FROZEN_FRESH` / `FROZEN` | bot **不卖, 不评估**, 等 24h 或解冻 |
| HOLD / MARGINAL / SOFT_NEGATIVE / AT_TARGET | 写状态, 等用户在 dashboard 操作 |

### 4.5 急跌检测算法 (`_drop_duration_minutes`)

```
1. 当 cur_price ≤ stop_price(entry) 时触发
2. 拉 Polymarket prices-history (interval=1h, fidelity=1) — 60 个 1-min 点
3. 从最新往回扫, 找最近一个 price > stop 的点 (= 跌破前最后高点)
4. duration_minutes = now − (跌破前最后高点的下一个点的 ts)
5. duration < 30min  → 急跌 → FROZEN_FRESH (冻结)
   duration ≥ 30min  → 慢跌 → SLOW_DROP (卖)
   API 失败 / 无数据 → 当作慢跌 (保守卖)
```

### 4.6 冻结状态管理

DB 列:
- `freeze_until` (TEXT, ISO 时间) — 冻结到期
- `freeze_stop_price` (REAL) — 触发冻结时的 stop_price

helpers (db.py):
- `set_freeze_until(token_id, until_iso, stop_price)`
- `clear_freeze(token_id)` — 价回弹时调

监控循环每 3 分钟检查:
- 冻结期内 → 不评估, 不卖 (保护机制)
- 价回到 `entry − 10pp` 以内 → `clear_freeze()` 解冻, 继续走正常评估
- 冻结期满 + 价仍 < stop → 卖 (`FROZEN_EXPIRED`)
- 冻结期满 + 价已回到 entry−10pp 内 → 解冻, 走正常

## 五、REEVAL Prompt v5.2 详解

### 5.1 触发时机

Dashboard 点黄 badge "🔄 重评" → `/api/reeval_prompt` → `build_reeval_prompt(meta, cur_price, days)` → 返回 prompt → 用户复制 → Claude.ai Research。

### 5.2 核心方法论: 盲评优先

| 阶段 | 任务 | 看持仓信息? |
|---|---|---|
| **A. 盲评** | 完全从零形成判断 (Bull/Bear 框架) | 🔒 不看 |
| **B. 24h 增量** | 微调 q | 🔒 不看 |
| **C. 对比** | 比对 q_today / 原 q / 入场 / 市价 | 看 |
| **D. 决策** | hold / update_q / exit + IRR | 看 |

### 5.3 研究深度 (按距结算)

| 距结算 | 盲评回溯窗口 | 总研究时间 |
|---|---|---|
| > 60 天 | 60 天 + 全部历史 | 20-30 min |
| 30-60 天 | 30 天 + 关键历史 | 15-25 min |
| 7-30 天 | 14 天 + 关键历史 | 12-18 min |
| < 7 天 | N×1.5 天 (N=距结算) | 8-12 min |

### 5.4 仓位详情 — YES/NO 双向标注 (强制)

每个百分比同时显示 `X% YES | Y% NO`:

```
━━━ 公开市场信息 (盲评可用) ━━━
市场: <market title>
Resolution 规则 (Polymarket 市场原文):
<完整 1500+ 字 description 从 Gamma API 抓取>
```

## 六、DISCOVERY Prompt (新仓研究)

### 6.1 触发

Dashboard 点 "🤖 复制给Claude" → `/api/full_prompt` → DISCOVERY 模板 + 最近扫描结果。

### 6.2 关键流程

每个候选要 Claude:
1. **Bull / Bear 视角** (各 1-2 个权威来源)
2. **原始估算 raw**
3. **校准后** = `市场价 + 0.5 × (raw − 市场价)` (五折)
4. **校准后 Edge** = calibrated − cur
5. **执行成本** = taker 滑点 + 双边 2.2pp
6. **实际可获取 Edge** = 校准 edge − 执行成本

### 6.3 推荐门槛

| 模式 | 校准 edge 门槛 |
|---|---|
| 标准 | ≥ **8pp** |
| 中范围 | ≥ **11pp** |
| 大范围 | ≥ **14pp** |

### 6.4 价格区间不对称

| 模式 | 价格范围 |
|---|---|
| 标准 | 8% — 92% |
| 中 | 5% — 95% |
| 大 | 3% — 97% |

低端宽因 < 15% 是 longshot 反向操作甜区。

### 6.5 年化 IRR

```
IRR = (calibrated − p) / p × 365 / days × 100%
< 30%: 不推荐 | 30-100%: 可接受 | > 100%: 甜区
```

## 十一、关键公式速查表

| 概念 | 公式 |
|---|---|
| Edge | `(q − p) × 100` (pp) |
| 单仓 PnL$ | `(cur_price − avg_price) × size` |

| 资产组合 | `Σ cur_price × size + 现金 USDC` |

| 年化 IRR | `(q − p) / p × 365 / days × 100%` |
| 实际可获取 edge | `校准 edge − 执行成本` |
| 浮盈/亏 % | `(cur − entry) / entry × 100` |
| **stop_price 分档** ⭐ v5 | `≥50¢→entry-0.25 / 30-50¢→entry-0.18 / 15-30¢→entry-0.10 / <15¢→None` |
| **绝对兜底触发** ⭐ v5 | `cur / entry < 0.40` (亏 60%) |
