# Polymarket v5.1

*[Chinese version](README.zh.md)*

A Polymarket prediction-market trading bot with an edge-based decision engine, two take-profit rules, a 3-tier stop loss, and a local Flask dashboard. Probability calibration is performed manually via Claude.ai; monitoring and execution are automated.

## v5.1 Auto Take-Profit (highest priority)

Two unconditional auto-sell rules that bypass edge re-eval and freeze states:

1. **Price take-profit** — `cur_price ≥ 90¢` → bot sells the full position. **Why 90¢**: marginal upside above 90¢ is low (annualized ~25%), liquidity thins out, and UMA resolution risk grows.
2. **Floating P&L doubled** — `(cur − entry) / entry ≥ +100%` → bot sells the full position. **Why +100%**: doubling is the psychological and mathematical "already made 1×" anchor; anything beyond is paper that can evaporate. +70% is too tight and would have prematurely closed big winners.

Either rule triggers a single full-size sell (no scaling out). Each rule uses its own `executed_action` tag to avoid re-firing.

## v5 Stop Loss (key change vs v4)

3-tier system replacing v4's single −25pp DISASTER threshold:

1. **Slow Hard Stop** — `cur_price ≤ stop_price(entry)` AND drop took > 30 min → bot auto-sells.
2. **Fast Drop Freeze** — same threshold hit but drop took < 30 min → bot freezes the position for 24 h (won't sell, won't take reeval signals). Auto-unfreezes if price recovers to `entry − 10pp`. After 24 h still below stop → bot auto-sells.
3. **Absolute Floor** — `cur_price / entry < 40%` (lost 60% of cost) → bot auto-sells regardless of state.

Where `stop_price(entry)` is tier-based:

| entry | drop | example stop |
|---|---|---|
| ≥ 50¢ | 25pp | 70¢ → 45¢ |
| 30 – 50¢ | 18pp | 40¢ → 22¢ |
| 15 – 30¢ | 10pp | 17¢ → 7¢ |
| < 15¢ | (none) | rule 3 only |

## Overview

The bot is a research-assist + monitor + execution stack rather than a fully algorithmic trader. Day-to-day workflow:

1. **Scan** — `scanner.py` queries the Polymarket Gamma API and produces a filtered list of candidate markets (35-tag whitelist + market filters).
2. **Research** — `prompts.py` renders DISCOVERY / REEVAL v5.2 prompts. The user pastes them into Claude.ai to obtain a calibrated target probability `q`.
3. **Position metadata** — the user enters `q`, entry price, confidence into the dashboard. State is stored in `position_meta` (SQLite).
4. **Monitor** — every 3 minutes, `monitor.py` fetches current positions and prices, evaluates two take-profit rules + 3-tier stop + TIME_STOP + edge-based logic per position.
5. **Act** — automatic sells: `TAKE_PROFIT_PRICE` / `TAKE_PROFIT_PNL` / `TIME_STOP` / `SLOW_DROP` / `FROZEN_EXPIRED` / `ABSOLUTE_FLOOR`. All other states surface in the dashboard for user-confirmed action.

## Setup

### Prerequisites
- macOS or Linux
- Python 3.10+
- A Polymarket account with a proxy wallet (Magic / GNOSIS_SAFE)

### Install
```bash
git clone https://github.com/RobinVico/polymarket.git
cd polymarket
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Configure
Copy `.env.example` to `.env` and fill in:

```
POLY_PRIVATE_KEY=0x...        # from https://polymarket.com/profile
POLY_FUNDER=0x...             # proxy wallet (NOT the EOA)
POLY_SIGNATURE_TYPE=1         # 1 = Magic / GNOSIS_SAFE
```

`POLY_API_KEY` / `POLY_API_SECRET` / `POLY_API_PASSPHRASE` are auto-generated by `py-clob-client` on first run; leave them empty.

> The `POLY_FUNDER` must be the proxy wallet, not the EOA. With `POLY_SIGNATURE_TYPE=1`, signing happens against the GNOSIS_SAFE owned by the EOA. Mixing these up produces silent signing failures.

### Run
Foreground:
```bash
source .venv/bin/activate
python3 main.py
```

Background:
```bash
source .venv/bin/activate
nohup python3 main.py > output.log 2>&1 &
```

Dashboard: <http://localhost:5051>

## Operations

### Restart after code change
```bash
pkill -f "main.py" 2>/dev/null
sleep 2
lsof -ti:5051 2>/dev/null | xargs kill -9 2>/dev/null
sleep 1
source .venv/bin/activate
nohup python3 main.py > output.log 2>&1 &
sleep 5
tail -10 bot.log
```

### Logs
- `bot.log` — application log (Python `logging`).
- `output.log` — stdout / stderr from `nohup`.

Both are gitignored.

### Database
SQLite at `v4.db` (filename kept for backward compatibility). Before any schema change:

```bash
cp v4.db v4.db.bak_$(date +%s)
```

`v4.db` and `v4.db.bak_*` are gitignored. Operational state is not version-controlled; rely on these timestamped copies plus the portfolio snapshot remote backup below.

### Portfolio Snapshot Remote Backup

`data/portfolio_snapshot.jsonl` is exported every 30 min by `scripts/auto_backup.sh` (via cron) and pushed to this GitHub repo. To restore on a new machine:

```bash
git pull origin main
bash scripts/restore_portfolio_snapshot.sh
```

`INSERT OR IGNORE` deduplicates by `ts` primary key, so the restore script is idempotent.
