## 一、项目架构总览

### 1.1 模块职责

| 模块 | 行数级别 | 职责 |
|---|---|---|
| `main.py` | ~30 | 入口: init DB → seed portfolio snapshot → 启动 monitor 线程 → 跑 Flask |

| `modules/monitor.py` | ~190 | 决策引擎, 3 分钟心跳, 自动止损执行 |

| `modules/tags.py` | ~200 | 22 个白名单 tag (3 个 tier) + 黑名单 |

### 1.2 关键技术决策 (不能改)

- 用 `py_clob_client_v2`, 不是 v1 (v1 有 order_version_mismatch bug)
- `POLY_SIGNATURE_TYPE=1` (GNOSIS_SAFE)
- `POLY_FUNDER` 必须是 proxy wallet, **不是** EOA
- 启动用 `nohup`, **不碰** launchd / systemd
- 数据库 schema 改动前先 `cp v4.db v4.db.bak_$(date +%s)`
- v3 在 `<sibling-v3-dir, not in this repo>` (frozen, 不能改)
- 重启前先看 `bot.log` 无报错才算成功

### 1.3 启动流程


## 二、用户日常使用流程

### 2.1 完整流程图

```
[扫描] → [研究] → [入场] → [监控] → [重评] → [平仓]
  ↑                                              │
  └────────────── 闲时回到扫描 ──────────────────┘
```

### 2.2 入场流程 (新仓位)

| 步 | 在哪 | 做什么 |
|---|---|---|
| 1 | Dashboard 顶部 "🎯 复制给Claude" | 点按钮,把 DISCOVERY prompt 复制到剪贴板 (含最近扫描结果) |
| 2 | Claude.ai | 贴上,Research 模式跑,等输出推荐 (可能 "无推荐") |
| 3 | Polymarket 网页 | 按 Claude 建议的方向手动买入 (bot 不自动下单) |
| 4 | Dashboard 持仓表 | 找到刚买的仓位,在 q 输入框填 Claude 给的"校准后估算"百分比 |
| 5 | 信心下拉框 | 选 高/中/低 (Claude 输出里有标),改了**自动保存** (不用点按钮) |
| 6 | 点 "保存" | 一次性把 TP + 信心都存进 DB |

### 2.3 日常监控

- **Dashboard 永远开着** (Mac mini 上),每 30s 自动刷新仓位/价格/盈亏
- **看 4 个 metric 卡**: 总盈亏 / 持仓数 / 总投入+总收入 / 资产组合+现金
- **看 🎯 事件中心**: 切换 实时榜/涨跌榜/现值榜,知道哪些仓位在动
- **看每个仓位的决策状态 badge**:
  - 🟢 `HOLD` — 继续放着
  - 🟡 `MARGINAL` / `SOFT_NEGATIVE` — 需要重评
  - 🔴 `CONFIRMED_NEGATIVE` / `AT_TARGET` / `BLACKSWAN_HEDGE` — 点 badge 走 sell
  - 灰 `PENDING` / `NO_META` — 还没填 q

### 2.4 重评流程

| 步 | 在哪 | 做什么 |
|---|---|---|
| 1 | 仓位右侧黄 badge "🔄 重评" | 点开 reeval-panel |
| 2 | "📋 复制 prompt" | 复制 v5.2 prompt (含 Resolution 规则全文 + YES/NO 双向价格) |
| 3 | Claude.ai Research | 贴上,等 Deep Research (8-30 min 看距结算) |
| 4 | 看 Claude 输出 | hold / update_q={X}% / exit |
| 5 | Dashboard 上选 | 按结果点 "维持 q" / 填新 q 点 "更新 q" / "建议清仓" |

### 2.5 平仓 (用户主动)

两种触发方式:
- **`monitor_state` 是红 badge** → 点 badge → 弹确认 → 走 executor.sell
- **重评 panel 里点 "建议清仓"** → 同上路径

Bot 永远不会**未经确认**清仓,除了:
- `DISASTER` (跌 ≥25pp): 自动卖
- `TIME_STOP` (≤2 天 + 漂移 <5pp): 自动卖

### 2.6 紧急情况

- **服务挂了**: 跑 §12.2 的重启流程
- **DB 损毁**: 跑 §12.3 的灾难恢复 (从 GitHub 拉 portfolio_snapshot.jsonl)
- **bot 自动卖出失败**: 看 `bot.log` 的 ERROR 行 (Polymarket API 偶尔返错,monitor 下个周期会重试)
- **想强制立即备份所有数据**: `bash scripts/auto_backup.sh`

## 四、决策引擎 (核心)

### 4.1 8 个状态 + 优先级

`modules/monitor.py:49 _evaluate_position()` — 每个仓位独立评估, **第一个匹配的状态胜出**:

| 优先级 | 状态 | 触发条件 | 动作 |
|---|---|---|---|
| 0 | `NO_META` | meta 行不存在 (没填过 q) | 跳过 |
| 1 | `TIME_STOP` | 距结算 ≤ 2 天 且 \|p − entry\| < 5pp | **自动卖** |
| 2 | `DISASTER` | entry − p ≥ 25pp (跌 25pp) | **自动卖** |
| 3 | `BLACKSWAN_HEDGE` | p ≥ 0.97 且距结算 > 1 天 | 等用户确认 |
| 4 | Edge-based | `edge = q − p` 看子状态 | 等用户确认 |

#### Edge-based 子状态 (优先级 4 内)

| 子状态 | 条件 | 含义 | 推荐动作 |
|---|---|---|---|
| `HOLD` | `edge > +2pp` | cur 价远低于 q | 持有 |
| `MARGINAL` | `−3pp ≤ edge ≤ +2pp` | 边缘,接近目标 | 观察,可重评 |
| `AT_TARGET` | `edge < −3pp` 且**未重评过 q** | 自然达标 | 建议清仓 |
| `SOFT_NEGATIVE` | `edge < −3pp` + 重评过,第 1 次翻负 | 警戒 | 重评 |
| `CONFIRMED_NEGATIVE` | `edge < −3pp` + 重评过 + 上次也负 | thesis 翻车 | 建议清仓 |

注: **`DUST_HOLD` 已在 2026-05-10 删除** (commit `044190f`)。<5 股仓位现在走正常逻辑,如果触发自动卖可能因 Polymarket 最小订单失败,bot.log 会 log warning,不影响其他仓位。

### 4.2 阈值常量

`modules/monitor.py` 顶部:

```python
HOLD_MIN_EDGE_PP = 2.0           # edge > +2pp = HOLD
SOFT_NEGATIVE_THRESHOLD_PP = -3.0 # edge < -3pp = 负领域
DISASTER_DROP_PP = 25.0          # 跌 ≥25pp 自动卖
TIME_STOP_DAYS = 2               # ≤2 天到结算
TIME_STOP_DRIFT_PP = 5.0         # 价格 vs 入场漂移 <5pp 视为没动
BLACKSWAN_PRICE = 0.97           # p ≥97% 黑天鹅减半
BLACKSWAN_MIN_DAYS = 1           # 距结算 >1 天才考虑
CHECK_INTERVAL = 180             # 心跳 3 分钟
SNAPSHOT_INTERVAL = 1800         # portfolio_snapshot 节流 30 分钟
```

### 4.3 自动 vs 手动

| 状态 | 谁执行 |
|---|---|
| `TIME_STOP` | **bot 自动 executor.sell()** |
| `DISASTER` | **bot 自动 executor.sell()** |
| 其他 6 个 | 写 `monitor_state` 字段, dashboard badge 显示, 等用户点 |

### 4.4 决策伪代码

```
for each 仓位 p:
    meta = get_position_meta(p.token_id)
    if not meta: return NO_META

    days_left = (end_date - now).days
    edge_pp = (q - cur_price) * 100

    if days_left ≤ 2 and abs(cur - entry) * 100 < 5:
        AUTO_SELL → TIME_STOP

    if (entry - cur) * 100 ≥ 25:
        AUTO_SELL → DISASTER

    if cur ≥ 0.97 and days_left > 1:
        return BLACKSWAN_HEDGE

    if edge_pp > +2: return HOLD
    elif -3 ≤ edge_pp ≤ +2: return MARGINAL
    else:
        if not meta.last_reeval_at: return AT_TARGET
        elif meta.last_q_update_with_negative_edge: return CONFIRMED_NEGATIVE
        else: return SOFT_NEGATIVE
```

## 五、REEVAL Prompt v5.2 详解

### 5.1 触发时机

用户在 dashboard 点黄 badge "🔄 重评" → `/api/reeval_prompt` → 调 `prompts.build_reeval_prompt(meta, cur_price, days_to_resolution)` → 返回完整 prompt → 用户复制 → 贴到 Claude.ai Research 模式。

### 5.2 核心方法论: 盲评优先 (blind-first)

v5.2 跟 v4.1 最大区别: **强制 Claude 4 阶段分析**:

| 阶段 | 任务 | 看不看持仓信息? |
|---|---|---|
| **A. 盲评** | 完全从零形成判断 (Bull/Bear 框架) | **🔒 看不到原 q / 入场 / 浮盈亏** |
| **B. 24h 增量** | 微调 q | 同上 (只看公开市场) |
| **C. 对比** | 比对 q_today vs 原 q vs 入场 vs 市价 | 现在才能看 |
| **D. 决策** | hold / update_q / exit + IRR 检查 | 都可以看 |

**🔒 锚定防御**: 模板里"我的持仓信息"区块前后用 🔒 emoji 框住,明确告诉 Claude **阶段 C 之前刻意忽略**。

### 5.3 研究深度 (按距结算天数动态调整)

| 距结算 | 盲评回溯窗口 | 总研究时间 |
|---|---|---|
| > 60 天 | 60 天 + 全部历史背景 | 20-30 min |
| 30-60 天 | 30 天 + 关键历史 | 15-25 min |
| 7-30 天 | 14 天 + 关键历史 | 12-18 min |
| < 7 天 | N×1.5 天 (N=距结算) | 8-12 min |

### 5.4 仓位详情 — YES/NO 双向标注 (强制)

**关键**: 每个百分比都同时显示 `X% YES | Y% NO`,绝不只显示单值。

样例 (实测输出):
```
━━━ 公开市场信息 (盲评可用) ━━━
市场: Will Trump say AI during Xi summit?
Resolution 规则 (Polymarket 市场原文):
<完整 1500+ 字 description 全文从 Gamma API 抓取>

当前市价: 69.5% YES | 30.5% NO
距结算: 12 天

━━━ 我的持仓信息 (阶段 C 之前不要看!) ━━━
🔒 [以下信息在阶段 C 之前请刻意忽略]
方向: 持有 YES (我赌 YES 兑现, YES 价格涨 = 我赚)
入场时间: 2026-05-06
入场价: 58.0% YES | 42.0% NO
我之前估的 q (YES 兑现概率): 83.0% YES | 17.0% NO
浮盈/亏: +19.8%
原始研究 confidence: medium
🔒 [以上信息在阶段 C 之前请刻意忽略]
```

### 5.5 Resolution 规则注入 (Gamma description)

- 模板原本写 "(请使用工具拉取该 Polymarket 市场完整规则文本)"
- 现在 `dashboard.py:reeval_prompt_route` 调 Gamma API 时**顺手抓 `description` 字段**
- 字段在 `modules/prompts.py:build_reeval_prompt` 里通过 `meta["_market_description"]` 注入
- **一个字都不掉**, Claude 不用再去 Polymarket 网站查规则

### 5.6 阶段 A/B/C/D 内部分析要求 (深度做, 输出精简)

- **A1-A7**: Resolution 规则拆解 / Bull/Bear (≥2 一手 URL) / 非事件检查 / q_raw + 校准 / Edge 计算
- **B1-B4**: 24h 新事件 + 单日 Δ + 反向论证
- **C1-C3**: q_today 对比 + 轨迹漂移 + **沉没成本自问** ("以当前市价新开同方向, 我愿不愿意?")
- **D**: 实际可获取 Edge / IRR / 阈值检查 (>+2pp 且 IRR>30%)

### 5.7 输出结构 (Claude 必须按此结构)

```
## 重评 YYYY-MM-DD
### A. 盲评
**q_raw = X% → q_calibrated = Y%**
理由 (覆盖 Bull 最强 + Bear 最强 + 关键机构状态)
Bull 强度 / Bear 强度
Edge: 校准后 ±A pp | 执行成本 Y pp | 实际可获取 ±B pp

### B. 24h 增量
**q_today = X% (q_raw_today X'% 校准后)**
24h 新事件 (有则列, 无则一句话)
调整理由

### C. 对比
表格: q_today / 原 q / 入场价 / 市价
沉没成本自问
轨迹漂移

### D. 决策
**[hold / update_q=X% / exit]**
实际可获取 Edge / IRR
理由 (1-2 句) / 风险 (1 句) / 下次重评
```

### 5.8 三种结果

| 结果 | 触发条件 | 用户在 dashboard 做 |
|---|---|---|
| **hold** | 24h 无新信号或 edge_new > +2pp | 点 "维持 q" |
| **update_q=X%** | 24h 有 ≥2 条新信息支持调整 | 填新 q 点 "更新 q" |
| **exit** | edge_new < -2pp / IRR < 30% / 重大反向事件 | 点 "建议清仓" |

## 六、DISCOVERY Prompt (新仓研究)

### 6.1 触发时机

用户在 dashboard 点 "🤖 复制给Claude" → `/api/full_prompt` → 返回 DISCOVERY 模板 + 最近扫描结果 → 用户贴到 Claude.ai。

### 6.2 关键流程

对每个候选市场要求 Claude:
1. **Bull 视角** (1-2 个权威来源)
2. **Bear 视角** (1-2 个权威来源)
3. **原始估算 raw**: xx%
4. **校准后**: `市场价 + 0.5 × (raw − 市场价)` (五折校准, 因为市场流动性已含信息)
5. **校准后 Edge** = calibrated − 当前价
6. **执行成本** = taker 滑点 + 双边 2.2pp
7. **实际可获取 Edge** = 校准 edge − 执行成本

### 6.3 推荐门槛 (按扫描宽度)

| 扫描模式 | 头部标题 | 校准 edge 门槛 |
|---|---|---|
| 标准 | `# 标准扫描结果` | ≥ **8pp** |
| 中范围 | `# 📊 中范围扫描结果` | ≥ **11pp** |
| 大范围 | `# ⚠️ 大范围扫描结果` | ≥ **14pp** |

实际可获取 edge < 门槛 − 3pp 也说 "无推荐"。范围越宽要 edge 越大 (滑点高)。

### 6.4 价格区间不对称 (v4.1)

| 模式 | 价格范围 |
|---|---|
| 标准 | 8% — 92% |
| 中 | 5% — 95% |
| 大 | 3% — 97% |

低端宽因 < 15% 是 longshot 反向操作甜区 (Polymarket 实证: <10% 实际兑现 ~14%),Tier 3 类别优先考虑卖 NO。

### 6.5 年化 IRR 强制

```
IRR = (calibrated - p) / p × 365 / days × 100%
```

| IRR | 决定 |
|---|---|
| < 30% | **不推荐** (机会成本不划算) |
| 30 — 100% | 可接受 |
| > 100% | 甜区, 优先 |

### 6.6 Cluster 检查

多市场赌同一底层事件时,只推校准 edge 最大的那一个,避免分散下注。

### 6.7 Resolution 文本歧义警告

扫描报告标注 `⚠️ resolution 简略 (<200字)` 或 `⚠️ resolution 复杂 (>800字)`。歧义到无法判断时选"无推荐",不要硬猜。**LLM 预测失败最主要原因**。

## 七、Confidence 三档机制详解

### 7.1 三档定义 + REEVAL prompt 影响

`modules/prompts.py:327-333`:

```python
confidence = meta.get("original_confidence") or "medium"  # 默认 medium
if confidence == "high":   exception_clause = "必须有新信息支持任何 q 调整, 无例外"
elif confidence == "medium": exception_clause = "允许 ≤3pp 下调 q 不需新信息"
else: # low                 exception_clause = "允许 ≤5pp 下调 q 不需新信息"
```

| 信心档 | 意义 | Claude 在 reeval 时被允许的动作 |
|---|---|---|
| **高 (high)** | 你对原研究极有信心 | **零自主调整**, 必须举出新信息 (新闻/数据/对手动作) |
| **中 (medium)** ⭐ 默认 | 中等信心 (大多数情况) | ≤3pp 元认知微调允许 |
| **低 (low)** | 你也不太确定 | ≤5pp 元认知调整允许 |
| **空 (未选)** | NULL → 当 medium 处理 | 同 medium |

**注意**: 这只限制**下调** (让 edge 缩),**上调没有限制**。

### 7.2 数据流

```
用户在下拉框选 高/中/低
      ↓ onchange="saveConf()"
POST /api/update_confidence  ← 单字段 UPDATE,不动其他字段
      ↓
db.update_confidence(token_id, value)
      ↓
position_meta.original_confidence = 'high' / 'medium' / 'low' / NULL
      ↓
点重评 → build_reeval_prompt 读 meta.original_confidence
      ↓
prompt 里塞 exception_clause + confidence_label
```

**关键**: dropdown 改了**立刻保存**,toast 显示 "信心已保存: 高"。不需要点保存按钮。

### 7.3 默认值演变

| 时间 | 默认 | 原因 |
|---|---|---|
| 初期 | `high` | 保守起见, 不允许 Claude 自由调 |
| 2026-05-11 起 | **`medium`** | 实际使用发现 high 太严, Claude 即使有合理判断也不敢调 |

```sql
CREATE TABLE portfolio_snapshot (
```

## 十一、关键公式速查表

| 概念 | 公式 |
|---|---|
| Edge | `(q − p) × 100` (pp) |
| 单仓 PnL$ | `(cur_price − avg_price) × size` |
| 总盈亏 | `Σ 单仓 PnL$` |
| 总投入 | `Σ avg_price × size` |
| 持仓现值 | `Σ cur_price × size` |
| 资产组合 | `持仓现值 + 现金 USDC` |
| 校准估算 | `market_price + 0.5 × (raw − market_price)` |
| 年化 IRR | `(q − p) / p × 365 / days × 100%` |
| 执行成本估算 | `taker 滑点 + 双边 fee (~2.2pp)`,不知按 3pp |
| 实际可获取 edge | `校准 edge − 执行成本` |
| YES 价格 (从持有侧) | side==YES: `price`; side==NO: `1 − price` |
| NO 价格 (从持有侧) | side==YES: `1 − price`; side==NO: `price` |
| 浮盈/亏 % | `(cur − entry) / entry × 100` |
