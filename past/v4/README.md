# Polymarket v4

*[中文版](README.zh.md)*

A Polymarket prediction-market trading bot with an edge-based decision engine and a local Flask dashboard. Probability calibration is performed manually via Claude.ai; monitoring and execution are automated.

## Overview

The bot is a research-assist + monitor + execution stack rather than a fully algorithmic trader. Day-to-day workflow:

1. **Scan** — `scanner.py` queries the Polymarket Gamma API and produces a filtered list of candidate markets (tag whitelist + market filters).
2. **Research** — `prompts.py` renders DISCOVERY / REEVAL prompts. The user pastes them into Claude.ai to obtain a calibrated target probability `q`.
3. **Position metadata** — the user enters `q`, entry price, and other meta into the dashboard. State is stored in `position_meta` (SQLite).
4. **Monitor** — every 3 minutes, `monitor.py` fetches current positions and prices, computes `edge = q − p` per position, and writes a decision state.
5. **Act** — only `DISASTER` and `TIME_STOP` trigger automatic sells via `executor.py` (py-clob-client v2). All other states surface in the dashboard for user-confirmed action.

## Architecture

| Path | Role |
|---|---|
| `main.py` | Entry point. Initializes SQLite, starts monitor thread, runs Flask on `0.0.0.0:5051`. |
| `modules/dashboard.py` | Flask UI and HTTP routes. Renders positions, decisions, scan results, prompts. |
| `modules/monitor.py` | 9-state edge-based decision engine. 180-second heartbeat. |
| `modules/scanner.py` | Polymarket Gamma scanner with a `FILTERS` dictionary (volume, end date, price band, etc.). |
| `modules/executor.py` | py-clob-client v2 wrapper for fetching positions and submitting sell orders. |
| `modules/db.py` | SQLite schema + `position_meta` CRUD + event log. |
| `modules/prompts.py` | DISCOVERY and REEVAL prompt templates rendered for Claude.ai. |
| `modules/tags.py` | 22-tag whitelist used by the scanner. |

## Decision Engine

Heartbeat: 180s. Each position is evaluated independently. Priorities (first match wins):

1. **`NO_META`** — position has no metadata row; skip.
2. **`DUST_HOLD`** — `size < 5` shares; hold to settlement.
3. **`TIME_STOP`** *(auto-sell)* — `days_to_settle ≤ 2` and `|p − entry| < 5pp`. Idle position near settlement.
4. **`DISASTER`** *(auto-sell)* — `entry − p ≥ 25pp`. Hard drawdown stop.
5. **`BLACKSWAN_HEDGE`** — `p ≥ 0.97` and more than 1 day to settle. Surface for manual partial.
6. **Edge-based** (`edge = q − p`, in pp):
   - `HOLD` — `edge > +2pp`. Current price well below target; keep position.
   - `MARGINAL` — `−3pp ≤ edge ≤ +2pp`. Borderline.
   - `edge < −3pp` (current price above target):
     - `AT_TARGET` — user never adjusted `q`. Treat as natural target hit; recommend close.
     - `SOFT_NEGATIVE` — user adjusted `q`, first time edge is negative.
     - `CONFIRMED_NEGATIVE` — user adjusted `q`, second consecutive negative reading. Recommend close.

Only `DISASTER` and `TIME_STOP` execute sells without user confirmation. Everything else writes to `monitor_state` for the dashboard to render.

Threshold constants live at the top of `modules/monitor.py`.

## Setup

### Prerequisites
- macOS or Linux
- Python 3.10+
- A Polymarket account with a proxy wallet (Magic / GNOSIS_SAFE)

### Install
```bash
git clone https://github.com/RobinVico/polymarket.git
cd polymarket/v4
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Configure
Copy `.env.example` to `.env` and fill in:

```
POLY_PRIVATE_KEY=0x...        # exported from https://polymarket.com/profile
POLY_FUNDER=0x...             # proxy wallet address (NOT the EOA)
POLY_SIGNATURE_TYPE=1         # 1 = Magic / GNOSIS_SAFE
```

`POLY_API_KEY` / `POLY_API_SECRET` / `POLY_API_PASSPHRASE` are generated by `py-clob-client` on first run; leave them empty.

> The `POLY_FUNDER` must be the proxy wallet, not the EOA. With `POLY_SIGNATURE_TYPE=1`, signing happens against the GNOSIS_SAFE owned by the EOA. Mixing these up produces silent signing failures.

### Run
Foreground:
```bash
source .venv/bin/activate
python3 main.py
```

Background:
```bash
source .venv/bin/activate
nohup python3 main.py > output.log 2>&1 &
```

Dashboard: <http://localhost:5051>

## Operations

### Restart after a code change
Code changes do not hot-reload. Use:

```bash
pkill -f "main.py" 2>/dev/null
sleep 2
lsof -ti:5051 2>/dev/null | xargs kill -9 2>/dev/null
sleep 1
source .venv/bin/activate
nohup python3 main.py > output.log 2>&1 &
sleep 5
tail -10 bot.log
```

### Logs
- `bot.log` — application log (Python `logging`).
- `output.log` — stdout / stderr from `nohup`.

Both are gitignored.

### Database
SQLite at `v4.db`. Before any schema change:

```bash
cp v4.db v4.db.bak_$(date +%s)
```

`v4.db` and `v4.db.bak_*` are gitignored. Operational state is not version-controlled; rely on these timestamped copies.
